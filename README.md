# Лабораторная работа 1. Перемножение матриц на CPU и GPU с использованием CUDA

## Общая информация

**Дисциплина:** Высокопроизводительные вычисления  
**Тема работы:** Перемножение матриц  
**Язык реализации:** C++ / CUDA  
**Среда выполнения:** Google Colab с NVIDIA Tesla T4

## Цель работы

Реализовать алгоритм умножения двух квадратных матриц на CPU и на GPU с использованием CUDA, сравнить время выполнения для разных размеров входных данных, проверить корректность полученных результатов и оценить ускорение GPU-реализации по сравнению с CPU-реализацией.

## Постановка задачи

Требовалось реализовать программу, которая:

- перемножает две матрицы размером от `100x100` до `2000x2000`;
- содержит две функции умножения матриц:
  - на CPU;
  - на GPU с применением CUDA;
- выполняет проверку корректности результатов;
- измеряет время вычисления;
- проводит серию экспериментов для матриц разных размеров;
- вычисляет ускорение GPU относительно CPU;
- сохраняет результаты в виде таблицы и графиков.

## Почему работа выполнялась в Google Colab

Мой ноутбук не имеет совместимой NVIDIA-видеокарты, а технология CUDA поддерживается только GPU NVIDIA. Поэтому выполнение и тестирование CUDA-части были перенесены в **Google Colab**, где доступен GPU **Tesla T4**.

В курсе *«Параллельные вычисления»* в 7 семестре бакалавриата аналогичный способ уже использовался для проверки программ с CUDA. В данной работе был применён тот же принцип: исходный код написан на C++/CUDA, а измерения выполнены в среде Google Colab с включённым GPU.

## Описание реализации

Программа состоит из двух основных реализаций умножения матриц:

### 1. Умножение на CPU

CPU-версия реализована с помощью тройного цикла. Для более эффективного доступа к памяти использован порядок циклов `i-k-j`, а не классический `i-j-k`.

Идея работы:

- фиксируется строка матрицы `A`;
- фиксируется элемент этой строки;
- затем происходит проход по соответствующей строке матрицы `B`;
- накопленная сумма записывается в результирующую матрицу `C`.

Такой порядок циклов уменьшает число неэффективных обращений к памяти и ускоряет выполнение по сравнению с базовым вариантом.

### 2. Умножение на GPU

GPU-версия реализована через CUDA-ядро. В ней использован tiled-подход с применением shared memory (разделяемая память).

Основные шаги GPU-реализации:

1. Выделение памяти на устройстве.
2. Копирование матриц `A` и `B` из памяти CPU в память GPU.
3. Запуск CUDA-ядра.
4. Копирование результирующей матрицы `C` обратно на CPU.
5. Измерение времени выполнения.

## Что именно было распараллелено и почему

На GPU было распараллелено **вычисление элементов результирующей матрицы**.

Каждый поток CUDA вычисляет один элемент результата `C[row][col]`.
Для этого поток выполняет скалярное произведение:

- `row`-й строки матрицы `A`;
- `col`-го столбца матрицы `B`.

### Почему выбрано именно такое распараллеливание

Этот способ естественен для задачи умножения матриц, потому что:

- каждый элемент результирующей матрицы вычисляется независимо от остальных;
- между вычислениями разных элементов нет конфликтов записи;
- большое количество потоков может работать одновременно;
- структура задачи хорошо соответствует модели CUDA «один поток — одна независимая операция».

## Почему использовалась разделяемая память

Если каждый поток будет каждый раз читать данные только из глобальной памяти GPU, это приведёт к большим задержкам. Поэтому в CUDA-ядре использовалась shared memory.

Матрицы разбиваются на блоки (`tiles`) размером `16x16`. На каждом шаге вычисления:

- часть матрицы `A` загружается в shared memory;
- часть матрицы `B` загружается в shared memory;
- потоки блока используют эти данные многократно;
- затем переходят к следующему тайлу.

Преимущество такого подхода состоит в том, что число обращений к медленной глобальной памяти уменьшается, а производительность GPU-ядра возрастает.

## Проверка корректности

Корректность GPU-результата проверялась путём сравнения с результатом CPU-вычисления.

Для проверки использовалась допустимая погрешность для типа `float`.
Вычислялась максимальная абсолютная ошибка `max_error`.

Во всех проведённых экспериментах результаты CPU и GPU совпали с допустимой погрешностью:

- `Correct = YES` для всех размеров матриц;
- максимальная ошибка не превысила `0.000034`.

Это и подтверждает корректность реализации.

## Методика проведения экспериментов

Эксперименты проводились для следующих размеров квадратных матриц:

- `100x100`
- `250x250`
- `500x500`
- `1000x1000`
- `1500x1500`
- `2000x2000`

Для повышения стабильности измерений каждый тест выполнялся 3 раза, после чего вычислялось среднее время:

- `CPU avg ms` — среднее время CPU;
- `GPU kernel ms` — среднее время выполнения только CUDA-ядра;
- `GPU total ms` — среднее полное время GPU-вычисления с учётом копирования данных;
- `speedup_total = CPU / GPU total`.

В итоговой таблице использовалось именно полное GPU-время, поскольку оно более честно отражает реальную стоимость вычисления на GPU.

### Анализ результатов

По результатам экспериментов можно сделать следующие выводы:

1. **Во всех тестах GPU показал более высокую производительность, чем CPU.**
2. **С ростом размера матрицы ускорение возрастает.** Это объясняется тем, что при больших объёмах вычислений GPU использует массовый параллелизм значительно эффективнее.
3. Для матрицы `2000x2000` ускорение составило примерно **110.39 раза**.
4. На малых матрицах выигрыш меньше, так как влияние накладных расходов на запуск ядра и передачу данных заметнее.
5. На больших размерах преимущество GPU становится особенно выраженным.
